## What is Caddy?

**Caddy** is a powerful, modern web server written in Go that stands out for its **automatic HTTPS**, **zero-configuration** approach, and **human-readable configuration**. Unlike traditional web servers that require complex setup procedures, Caddy is designed to work out of the box with sensible defaults while remaining highly configurable for advanced use cases.

### Key Philosophy

Caddy was built with the philosophy that **HTTPS should be the default**, not an afterthought. It automatically obtains and renews SSL/TLS certificates from Let's Encrypt and other Certificate Authorities, making secure web serving accessible to everyone without the traditional complexity.

### ğŸ” **Understanding IP Addresses - The Basics**

Before we explain the traffic flow, you need to understand the different types of IP addresses:

### ğŸ“‹ **Quick Reference: Which IP Goes Where?**

| **What You're Configuring** | **Which IP to Use** | **Example** |
|------------------------------|---------------------|-------------|
| **Router Port Forwarding** | Internal/Private IP | `192.168.1.100` |
| **DNS A Record** | External/Public IP | `203.0.113.42` |
| **Caddyfile (Public Site)** | Domain Name | `mywebsite.com` |

**ğŸ’¡ Pro Tip**: Set a **static IP reservation** in your router for your server computer. This prevents your internal IP from changing and breaking port forwarding!

#### **ğŸ  Internal IP Address (Private IP)**
- **What it is**: The IP address your computer has on your home network
- **Examples**: `192.168.1.100`, `10.0.0.50`, `172.16.1.25`
- **Who can access it**: Only devices on your home network (your phone, laptop, etc.)
- **Used for**: Port forwarding rules in your router

#### **ğŸŒ External IP Address (Public IP)**
- **What it is**: The IP address your internet provider gives your home
- **Example**: `203.0.113.42` (changes periodically unless you pay for static IP)
- **Who can access it**: Anyone on the internet
- **Used for**: DNS records, telling people how to reach your server

#### **ğŸ’» Localhost**
- **What it is**: A special address that means "this computer"
- **Address**: Always `127.0.0.1` or `localhost`
- **Who can access it**: Only the computer itself
- **Used for**: Testing and internal services

### ğŸ“‹ **Step-by-Step: How Internet Traffic Reaches Your Caddy Server**

#### **Step 1: ğŸ‘¤ Someone Types Your Website Address**
- User types `mywebsite.com` in their browser
- Browser needs to find out what IP address that domain points to

#### **Step 2: ğŸ” DNS Lookup (Finding Your IP Address)**
- **Simple explanation**: Like looking up a phone number in a phone book
- Browser asks DNS servers: "What IP address is mywebsite.com?"
- DNS responds: "It's at 203.0.113.42" (your external IP)

#### **Step 3: ğŸŒ Optional: CDN (Content Delivery Network)**
- **What it does**: Stores copies of your website closer to users
- **Popular services**: Cloudflare, AWS CloudFront
- **Benefit**: Makes your website load faster
- **Note**: Not required for basic setups

#### **Step 4: ğŸŒ Internet Routing**
- User's internet provider sends the request across the internet
- Request travels through multiple routers to reach your home
- **You don't need to configure anything here**

#### **Step 5: ğŸ  Your Router Receives the Request**
- Request arrives at your home router on ports 80 (HTTP) or 443 (HTTPS)
- **This is where port forwarding comes in!**

### ğŸ”§ **Port Forwarding - The Critical Step**

**This is the part most people get confused about!**

#### **What is Port Forwarding?**
- Your router receives internet traffic but doesn't know which computer should handle it
- Port forwarding tells your router: "Send web traffic to this specific computer"

#### **What You Need to Configure:**

1. **External Port**: `80` (HTTP) and `443` (HTTPS) - these are standard
2. **Internal IP**: Your computer's IP address on your home network
3. **Internal Port**: Usually the same (`80` and `443`)

#### **How to Find Your Computer's Internal IP:**

**Windows:**
```cmd
ipconfig
```
Look for "IPv4 Address" - usually starts with `192.168.` or `10.0.`

**Mac/Linux:**
```bash
ip addr show
# or
ifconfig
```

**Example Port Forwarding Rule:**
```
External Port: 80 â†’ Internal IP: 192.168.1.100 â†’ Internal Port: 80
External Port: 443 â†’ Internal IP: 192.168.1.100 â†’ Internal Port: 443
```

#### **Step 6: âš¡ Your Computer Runs Caddy**
- Caddy listens on ports 80 and 443
- When traffic arrives, Caddy handles the request
- Caddy can serve files or forward requests to other programs

#### **Step 7: ğŸ¯ Your Application (Optional)**
- If you're running a web app (like a blog or store), Caddy forwards requests to it
- The app processes the request and sends a response back
- Caddy sends the response back to the user

Open Command Prompt/Terminal in your website folder and run:


Open your browser and go to: `http://localhost:8080`

### **ğŸ¤” Still Confused About Networking?**

**Common beginner issues:**

1. **"My website works locally but not from the internet"**
   - Check your port forwarding settings
   - Make sure you're using your domain name, not localhost
   - Verify your DNS records point to your external IP
   - **Check for CGNAT**: Some ISPs use "Carrier-Grade NAT" which blocks port forwarding. Call your ISP if port forwarding doesn't work.

2. **"I get certificate errors"**
   - Make sure your domain points to your server
   - Check that ports 80 and 443 are open
   - Wait a few minutes for certificates to be issued

3. **"Caddy won't start"**
   - Check if another program is using ports 80 or 443
   - Make sure you're running as administrator (Windows) or with sudo (Linux)
   - Verify your Caddyfile syntax

4. **"Port forwarding stopped working"**
   - Your computer's internal IP probably changed
   - Set up **DHCP reservation** in your router to keep the same internal IP
   - Update port forwarding rules with the new internal IP if needed
